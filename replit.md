# Workspace

## Overview

pnpm workspace monorepo using TypeScript. Each package manages its own dependencies.

## Stack

- **Monorepo tool**: pnpm workspaces
- **Node.js version**: 24
- **Package manager**: pnpm
- **TypeScript version**: 5.9
- **API framework**: Express 5
- **Database**: PostgreSQL + Drizzle ORM
- **Validation**: Zod (`zod/v4`), `drizzle-zod`
- **API codegen**: Orval (from OpenAPI spec)
- **Build**: esbuild (CJS bundle)

## GharPlus Mobile App

GharPlus is a smart Indian household management mobile app built with Expo.

### Features
- **Home Dashboard** тАФ Budget tracker, quick actions grid, stats (pantry count, grocery count, meal plan), today's meals
- **AI Meal Planner** тАФ 7-day Indian meal plan generated by Gemini AI based on region & diet preference
- **Pantry Manager** тАФ Track pantry items with status (Good/Low/Out), AI fridge scanner via image upload, manual add
- **Grocery Order** тАФ Shopping list with AI categorization, Smart Swap AI suggestions for healthier/cheaper alternatives
- **Minani AI Chat** тАФ Floating AI household assistant for recipes, tips, home remedies
- **Settings** тАФ Family profile, budget reset, app info

### Tech Stack
- **Framework**: Expo / React Native (Expo Router)
- **AI**: Google Gemini (via Replit AI Integrations, proxied through Express API)
- **State**: React Context + AsyncStorage (no backend DB for first build)
- **Theme**: Dark mode, warm orange (#E67E22) primary accent, #141414 background
- **Icons**: @expo/vector-icons (Feather)

### AI Routes (on API Server at /api/gharplus/*)
- `POST /api/gharplus/meal-plan` тАФ Generate 7-day meal plan
- `POST /api/gharplus/analyze-image` тАФ Analyze fridge/pantry image
- `POST /api/gharplus/categorize` тАФ Categorize grocery items
- `POST /api/gharplus/smart-swap` тАФ Get smart grocery swap suggestion
- `POST /api/gharplus/minani` тАФ Chat with Minani AI assistant

### Setup Flow
New users go through a 5-step onboarding: Welcome тЖТ Name тЖТ Region тЖТ Diet тЖТ Budget
After setup, routed to the main tabs (Home, Plan, Pantry, Order, Settings)

## TypeScript & Composite Projects

Every package extends `tsconfig.base.json` which sets `composite: true`. The root `tsconfig.json` lists all packages as project references. This means:

- **Always typecheck from the root** тАФ run `pnpm run typecheck` (which runs `tsc --build --emitDeclarationOnly`). This builds the full dependency graph so that cross-package imports resolve correctly. Running `tsc` inside a single package will fail if its dependencies haven't been built yet.
- **`emitDeclarationOnly`** тАФ we only emit `.d.ts` files during typecheck; actual JS bundling is handled by esbuild/tsx/vite...etc, not `tsc`.
- **Project references** тАФ when package A depends on package B, A's `tsconfig.json` must list B in its `references` array. `tsc --build` uses this to determine build order and skip up-to-date packages.

## Root Scripts

- `pnpm run build` тАФ runs `typecheck` first, then recursively runs `build` in all packages that define it
- `pnpm run typecheck` тАФ runs `tsc --build --emitDeclarationOnly` using project references

## Packages

### `artifacts/api-server` (`@workspace/api-server`)

Express 5 API server. Routes live in `src/routes/` and use `@workspace/api-zod` for request and response validation and `@workspace/db` for persistence.

- Entry: `src/index.ts` тАФ reads `PORT`, starts Express
- App setup: `src/app.ts` тАФ mounts CORS, JSON/urlencoded parsing, routes at `/api`
- Routes: `src/routes/index.ts` mounts sub-routers; `src/routes/health.ts` exposes `GET /health` (full path: `/api/health`)
- Routes: `src/routes/gharplus.ts` тАФ GharPlus AI endpoints at `/api/gharplus/*`
- Depends on: `@workspace/db`, `@workspace/api-zod`, `@google/genai`
- `pnpm --filter @workspace/api-server run dev` тАФ run the dev server

### `artifacts/gharplus` (`@workspace/gharplus`)

GharPlus Expo mobile app. Frontend-only with AI powered by API server.

- Entry: `app/_layout.tsx` тАФ root layout with providers (SafeAreaProvider, ErrorBoundary, QueryClientProvider, AppProvider, GestureHandlerRootView, KeyboardProvider)
- Navigation: `app/(tabs)/_layout.tsx` тАФ 5-tab layout (Home, Plan, Pantry, Order, Settings) with NativeTabs for iOS 26+ liquid glass
- Context: `contexts/AppContext.tsx` тАФ family profile, pantry, grocery, meal plan state with AsyncStorage persistence
- AI services: `services/gemini.ts` тАФ calls API server endpoints
- Components: `components/GharCard.tsx`, `components/GharButton.tsx`, `components/MinaniChat.tsx`

### `lib/db` (`@workspace/db`)

Database layer using Drizzle ORM with PostgreSQL. Exports a Drizzle client instance and schema models.

### `lib/api-spec` (`@workspace/api-spec`)

Owns the OpenAPI 3.1 spec (`openapi.yaml`) and the Orval config (`orval.config.ts`).
Run codegen: `pnpm --filter @workspace/api-spec run codegen`
